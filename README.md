# 🛡️ V2Ray Collector — نسخه گیت‌هاب

یک صفحه‌ی وب مستقل و زیبا برای **جمع‌آوری خودکار کانفیگ V2Ray از کانال‌های تلگرام**، **تزریق آی‌پی تمیز** و **ساخت لینک ساب (Subscription)** — قابل استفاده در موبایل، با فونت وزیر و رابط فارسی (RTL).

این نسخه دو حالت کار دارد:

1. **حالت مرورگری (GitHub Pages):** همه‌چیز داخل مرورگر شما اجرا می‌شود؛ داده‌ها در LocalStorage ذخیره می‌گردند.
2. **حالت کاملاً خودکار (GitHub Actions):** گیت‌هاب هر ۶ ساعت خودش کانفیگ‌ها را جمع می‌کند و یک **لینک ساب دائمی** می‌سازد.

---

## 📁 ساختار پروژه

```
.
├── index.html              صفحه‌ی اصلی
├── assets/
│   ├── styles.css          استایل
│   ├── engine.js           موتور پارس/تزریق کانفیگ
│   └── app.js              منطق رابط
├── data/
│   └── config.json         کانال‌ها و آی‌پی‌های تمیز (برای حالت خودکار)
├── scripts/
│   └── collect.py          اسکریپت جمع‌آوری (GitHub Actions)
├── .github/workflows/
│   └── collect.yml         زمان‌بندی خودکار
└── sub/                    خروجی: sub.txt (لینک ساب) + raw.txt + report.json
```

---

## 🚀 راه‌اندازی روی GitHub Pages

1. یک ریپازیتوری جدید بساز (مثلاً `v2ray-collector`).
2. همه‌ی فایل‌های این پوشه را در آن آپلود/پوش کن.
3. به **Settings → Pages** برو، در Source گزینه‌ی **Deploy from a branch** و شاخه‌ی `main` / پوشه‌ی `/(root)` را انتخاب کن.
4. بعد از چند دقیقه آدرس زیر فعال می‌شود:

   ```
   https://USERNAME.github.io/REPO/
   ```

حالا می‌توانی از خود صفحه کانال اضافه کنی، آی‌پی تمیز بدهی و کانفیگ بسازی.

---

## 🤖 حالت کاملاً خودکار (پیشنهادی) — لینک ساب دائمی

اگر می‌خواهی خودِ گیت‌هاب کانفیگ‌ها را جمع کند و لینک ساب ثابت بدهد:

1. فایل `data/config.json` را ویرایش کن:

   ```json
   {
     "channels": ["free_v2ray_collector", "another_channel"],
     "clean_ips": ["104.18.0.1:2053", "104.18.0.2:8443"],
     "per_channel": 3,
     "only_tls": false
   }
   ```

2. تغییرات را Commit کن. این کار به‌صورت خودکار Action را اجرا می‌کند (یا از تب **Actions** دستی Run workflow بزن).
3. لینک ساب دائمی تو این خواهد بود:

   ```
   https://raw.githubusercontent.com/USERNAME/REPO/main/sub/sub.txt
   ```

4. همین لینک را در **هیدیفای** یا **V2rayNG** به‌عنوان **Subscription** اضافه کن.

> Action طبق `cron: "0 */6 * * *"` هر ۶ ساعت اجرا می‌شود. برای تغییر فاصله، فایل `.github/workflows/collect.yml` را ویرایش کن.

### ⚙️ فعال‌سازی دسترسی نوشتن برای Action
اگر Action نتوانست push کند، به **Settings → Actions → General → Workflow permissions** برو و **Read and write permissions** را فعال کن.

---

## 📲 استفاده در هیدیفای / V2rayNG

- **Subscription:** لینک `sub.txt` (raw گیت‌هاب) را در بخش افزودن اشتراک وارد کن.
- **Import مستقیم:** محتوای `sub/raw.txt` کانفیگ‌های خام است؛ می‌توانی کپی و مستقیم Import کنی.
- در صفحه‌ی وب هم دکمه‌های «کپی لینک ساب»، «کپی کانفیگ خام» و «دانلود فایل ساب» موجود است.

---

## 🧹 منطق تزریق آی‌پی تمیز

آدرس اتصال هر کانفیگ با آی‌پی تمیز جایگزین می‌شود، ولی **SNI** و **Host** اصلی (دامنه‌ی واقعی) حفظ می‌گردد تا روی شبکه‌ی کلودفلر کار کند. از هر کانفیگ به‌تعداد آی‌پی‌های تمیز، کانفیگ جدید ساخته می‌شود (مثلاً ۵ کانفیگ × ۴ آی‌پی = ۲۰ کانفیگ).

## ❓ نکات و عیب‌یابی

| مشکل | راه‌حل |
|---|---|
| در مرورگر هیچ کانفیگی نمی‌گیرد | پروکسی CORS را در تب **تنظیمات** عوض کن |
| تست واقعی پینگ/سلامت می‌خواهم | از **نسخه‌ی شخصی (هاست + SQLite)** استفاده کن |
| Action نمی‌تواند Commit کند | Workflow permissions را روی Read and write بگذار |
| فقط کانفیگ امن می‌خواهم | گزینه‌ی «فقط TLS/Reality» را روشن کن (`only_tls: true`) |
